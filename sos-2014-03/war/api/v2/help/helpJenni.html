Ayuda de API Seville


Método GET


GET /Seville o GET /Seville/

Devuelve una lista en formato Json de todos los objetos almacenados en el DataStore.


GET /Seville/XXXX

Devuleve el objeto en formato Json del año especificado con "XXXX".

Ejemplo:

GET /Seville/1999

Objeto JSON :

{
"year": 2000,
"population": 500000,
"unemployed": 500,
"educationBudget": 1000,
"migrants": 2000,
"pib": 0.03
}

Si existe un objeto en el DataStore del año especificado en la url, lo devuelve en formato Json.
Además de devolver el objeto en formato Json, se muestra el código HTTP 200 OK.

En el caso de que no se encuentre en el DataStore el objeto con el año especificado en la url o se introduzca una cadena de caracteres,
se mostrará el código de error HTTP 404 Not found.

GET /Seville/XXXX/*

Devuelve el código de error HTTP 400 Bad Request.



Método POST


Añade en el DataStore un nuevo objeto.

POST /Seville o POST Seville/

Payload:

{
"year": 2000,
"population": 500000,
"unemployed": 500,
"educationBudget": 1000,
"migrants": 2000,
"pib": 0.03
}

Si el objeto se ha creado correctamente, devuelve el código HTTP 201 Created.
En el caso de que ya exista ese objeto en el DataStore, devolverá el código de error HTTP 409 CONFLICT.
A tener en cuenta los tipos de las propiedades de los datos:

year: number(long)
population: number(long)
unemployed: number(long)
educationBudget: number(double)
migrants: number(long)
pib : number(double)

Si el objeto no se ha parseado correctamente o en cualquier otro caso de error,
devolverá el código de error HTTP 409 Conflict

Cuando la url sea de tipo: /Seville/* se devolverá el código de error HTTP 400 Bad Request.


Método PUT


Actualiza el objeto del año especificado en el DataStore.

Cuando se quiere actualizar un año, ponemos en la url el año del objeto que se quiere actualizar.
En el Payload ponemos el objeto en formato Json a actualizar.

Ejemplo:

PUT /Seville/2000

Payload:{
"year": 2000,
"population": 500000,
"unemployed": 500,
"educationBudget": 1000,
"migrants": 2000,
"pib": 0.03
}

Se devuelve el código HTTP 200 OK.

Si los años coinciden, pero no existe ningún objeto en el DataStore de ese año,
se devolverá el código de error HTTP 404 Not Found.

Si el año que se le pasa a la url es diferente al año que aparece en el objeto en formato Json,
devolverá el código de error HTTP 401 Not Authorized.

No se permite ningún PUT si la url presenta uno de los siguientes formatos:
/Seville, /Seville/ o /Seville/XXXX/*.
Se devolverá el código de error HTTP 400 Bad Request.

Si el objeto no se ha parseado correctamente o en cualquier otro caso de error (se introduzca una cadena de caracteres),
devolverá el código de error HTTP 304 Not Modified.

Método DELETE


DELETE /Seville o /Seville/

Se borrará todos los objetos contenidos en el DataStore.
En el caso de que todo sea correcto, se devolverá el código HTTP 200 OK.

DELETE /Seville/XXXX, siendo XXXX el año del objeto a borrar.

Entonces se borrará el objeto con ese año. Si todo es correcto devolverá el código HTTP 200 OK.
En caso de que no exista en el DataStore el objeto del año especificado o se introduzca una cadena de caracteres,
se devolverá el código de error HTTP 404 Not Found.

Cuando la url sea de la siguiente forma: /Seville/XXXX/*,
se devolverá el código de error HTTP 400 Bad Request.
