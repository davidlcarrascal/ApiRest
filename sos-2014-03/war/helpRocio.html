Ayuda de API UniversitySeville


Método GET - Actualizar


Devuelve una tabla con todos los datos que contiene nuestro DataStore.


Si queremos información de un año en concreto, lo introduciremos en el campo year.
Y nos devolverá una tabla con los datos del año solicitado

Ejemplo:
Year: 2010
!!!!!!

Si el año del cual solicitamos información no se encuentra en la lista nos devolverá un error 404 – NOT FOUND

Método POST


/api/v1/universitySeville

Añade un nuevo dato a universitySeville.

Ejemplo:
Payload: {"year":2010,"enrolled":189089,"budget":19990000,"employability":2097,"studentMigrants":8900}

Si todo ha ido bien, devolverá 201 - CREATED.

Si ya existiera ese año , devolverá 409 - CONFLICT.

/api/v1/universitySeville/XXXX

No se permite hacer Post sobre ningun dato en concreto, se devolverá 400 - BAD REQUEST.


Método PUT


/api/v1/universitySeville

No esta permitido hacer Put sobre la lista, devolverá 400 - BAD REQUEST.

/api/v1/universitySeville/XXXX

Modifica los datos de universitySeville con identificador year : XXXX , si no contiene ese identificador devuelve un error 404 - NOT FOUND

Ejemplo:

/api/v1/universitySeville/2010

{"year":2010,"enrolled":000000,"budget":19990000,"employability":2097,"studentMigrants":8900}


Método DELETE


/api/v1/universitySeville

Borra la lista completa universitySeville

/api/v1/universitySeville/XXXX

Borra de universitySeville todos los datos relacionados con year: XXXX, si el año a borrar no existe devuelve un error 404 - NOT FOUND
