Ayuda de API SpainStd


Método GET


GET /SpainStd

Devuelve la lista que contiene todos los años en nuestra base de datos.


GET /SainStd/xxxx

Devuelve el objeto en JSON correspondiente al año escrito en "XXXX".

Ejemplo:

GET /SainStd/1999

Objeto JSON : {"year":1999,"population":30000021,"unemployed":1500000,"pib":2,"emigrants":234234,"educationBudget":55555}

Si todo ha ido bien devolverá el código de estado 200 OK.
En otro caso 404 NOT FOUND.



Método POST


Añade en la base de datos un nuevo objeto.

POST /SpainStd

payload: {"year":1999,"population":30000021,"unemployed":1500000,"pib":2,"emigrants":234234,"educationBudget":55555}

Si todo ha ido bien, devolverá 201 CREATED.
Si ya existe en la base de datos, devolverá 409 CONFLICT.
A tener en cuenta los tipos de los datos:

year: number
population: number
unemployed: number
pib : number
emigrants: number
educationBudget: number

No se permite ningun POST sobre un año en concreto /SpainStd/XXXX se devolverá 400 BAD REQUEST.


Método PUT


Actualiza un objeto de la base de datos.

No se permite ningún PUT sobre la lista /SpainStd se devolverá 400 BAD REQUEST.

Para actualizar un año, se introduce en la url a continuación de la lista y se introduce en el payload el objeto en JSON a actualizar. Debe haber concordancia entre el año del objeto pasado y el año escrito en la url en caso contrario se devolverá 404 NOT FOUND.

Ejemplo:

PUT /SpainStd/1999

payload: {"year":1999,"population":30000021,"unemployed":1500000,"pib":2,"emigrants":234234,"educationBudget":55555}


Método DELETE


DELETE /SpainStd

Se borrara toda la lista de Años. Si todo ha ido bien devolverá 204 NO CONTENT.

En caso contrario devolverá 404 NOT FOUND.

DELETE /SpainStd/XXXX, *Donde XXX= Año.

Se borrará el objeto con ese año. Si todo ha ido bien devolverá 204 NO CONTENT. En caso contrario devolverá 404 NOT FOUND.
