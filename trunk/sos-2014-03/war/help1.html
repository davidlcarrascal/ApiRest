Utilizaremos el metodo POST para:

  • Crear un nuevo servicio:(Nota: requiere un Centro Sanitario ya creado con el nica)

    http://sos-2013-g2.appspot.com/api/v1/services


    {"nica":"16003","id":"1234","name":"unidad de cuidados intensivos"}

  • Estados HTTP
    OK: se devolverá esta respuesta cuando creemos el servicio sin problemas.
    METHOD_NOT_ALLOWED: se devolverá esta respuesta cuando pasemos un parámetro por el enlace.
    BAD_REQUEST: se devolverá esta respuesta cuando intentemos crear un servicio que ya exista o no exista ningun centro con el NICA pasado como parametro.

Utilizaremos el metodo PUT para:

  • Actualizar un servicio: (Nota: hay que pasar en la url el servicio a modificar en formato JSON)

    http://sos-2013-g2.appspot.com/api/v1/services/{"nica": "18014", "id": "1234", "name":"unidad de cuidados intensivos"}


    {"nica": "18014", "id": "7777", "name":"unidad de cuidados intensivos"}

  • Estados HTTP
    OK: se devolverá esta respuesta cuando actualice con exito el servicio.
    METHOD_NOT_ALLOWED: se devolverá esta respuesta cuando no pasemos un parámetro por el enlace.
    NOT_FOUND: se devolverá esta respuesta cuando intentemos modificar un servicio que no exista.
    BAD_REQUEST: se devolverá esta respuesta cuando pasemos una cadena en la url no válida.

Utilizaremos el metodo DELETE para:

  • Borrar todos los servicios ofrecidos:

    http://sos-2013-g2.appspot.com/api/v1/services

  • Borrar un servicio ofrecido: (Nota: hay que pasar en la url el servicio a borrar en formato JSON)

    http://sos-2013-g2.appspot.com/api/v1/services//{"nica": "18014", "id": "1234", "name":"unidad de cuidados intensivos"}

  • Estados HTTP
    OK: se devolverá esta respuesta cuando borre con éxito el/los servicio/s.
    NOT_FOUND: se devolverá esta respuesta cuando intentemos borrar un servicio que no exista.
